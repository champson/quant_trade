# Quant Trade 设计文档

## 1. 目标

Quant Trade 是统一的 A 股研究工作台，覆盖四类操作：

1. 每日收盘复盘；
2. ETF、微盘股及后续策略的信号生成与回测；
3. 日线和分钟数据的下载、导入、校验与查询；
4. 可复现的研究产物、报告和运行记录。

原来的 `etf/`、`market_breadth/` 独立目录已移除。业务能力统一放在
`src/quant_trade/`，运行入口统一为 `qt`，生成文件统一进入 `artifacts/`。

## 2. 分层结构

```text
CLI / Dashboard
       │
       ├── pipelines       每日收盘等编排任务
       ├── services        策略信号、回测和数据更新用例
       ├── strategies      只负责从标准数据产生目标权重或信号
       ├── backtest        撮合、绩效计算和报告生成
       ├── reports         市场复盘报告
       └── data
            ├── providers  Tushare / Baostock / AkShare
            ├── router     重试、熔断和数据源降级
            ├── importers  ZIP及本地分钟目录导入
            └── storage    Parquet + DuckDB目录
```

策略不直接调用数据源、不自行管理缓存，也不自行绘图。新增策略只需实现标准信号和目标
权重接口，由公共服务完成数据读取、回测、成本建模和报告输出。

## 3. 数据设计

### 3.1 在线数据

数据路由按配置的优先级访问 Tushare、Baostock 和 AkShare。网络请求使用有限次数重试、
退避、熔断和可选数据源降级。缓存完整时不重复下载；数据来源与请求结果记录在 DuckDB。

### 3.2 分钟文件

同时支持 ZIP/CSV 和现有分钟目录。导入后的目录不再保留“股票/ETF/指数”的物理层级，
资产类型作为字段及 DuckDB 元数据保存：

```text
data/processed/minute/
  frequency=5min/
    symbol=000001_SZ/
      year=2025.parquet
```

源文件只读，输出使用 ZSTD Parquet。导入以文件状态和哈希断点续传，并记录输入行数、
过滤行数、时间覆盖和分区路径。5分钟数据可以向上聚合，但不能还原真实1分钟数据。

## 4. 工作流

### 4.1 收盘复盘

`qt daily run` 负责交易日判断、数据补齐、市场宽度计算、策略信号生成、报告落盘和运行
状态记录。单独调试复盘时使用 `qt review close`。

### 4.2 策略回测

```text
标准行情 → 策略目标权重 → 次日开盘撮合 → 绩效/交易统计 → 统一报告
```

收盘产生的目标权重只能在下一根可用行情的开盘执行。回测公共配置包含初始资金、佣金、
卖出印花税、滑点和无风险利率。策略配置中的 `benchmark` 若存在本地行情，会自动加入
比较；缺失时报告仍生成，但基准列显示为空。

每次 `qt strategy backtest <name>` 在 `artifacts/backtests/<name>/` 生成：

```text
report.html             自包含的完整阅读页面，图表以内嵌数据保存
report.md               可归档、可版本比较、可人工补充的报告
equity.png              净值、基准与回撤
monthly_returns.png     月度收益分布
equity.csv              每日权益
positions.csv           每日持仓权重
trades.csv              订单成交与成本
metrics.json            策略和基准的机器可读指标
```

HTML 是主要阅读入口，把策略配置、核心绩效、年度/月度表现、交易成本、可信度说明和图表
集成在一个页面中；Markdown 与它使用同一份计算结果，二者不允许分别计算指标。

## 5. 报告可信度原则

自动报告只陈述引擎实际计算或验证的内容。当前引擎已经控制“收盘信号不能在当日成交”，
并建模佣金、卖出印花税和滑点。以下事项尚未完整建模，必须在报告中显式标注：

- 涨跌停和停牌导致的无法成交；
- A股 T+1；
- 默认100股整数手约束；
- 财务数据公告时点；
- 退市股票和历史股票池导致的生存者偏差。

成交记录目前是订单流水，尚未配对为完整开平仓。因此单笔胜率、盈亏比、平均持有期、
盈利交易集中度不应伪造为自动指标。详细的目标字段与后续扩展顺序见
[`report.md`](../report.md)。

## 6. Dashboard

`qt dashboard` 是统一浏览入口：复盘、策略信号、回测和数据任务分为独立标签页。回测
标签页直接嵌入自包含的 `report.html`；旧回测产物没有 HTML 时，继续回退显示指标 JSON
和权益图。

## 7. 后续演进

1. 增加成交约束层，支持停牌、涨跌停、T+1和100股整数手；
2. 将订单配对为持仓周期，补齐交易胜率、盈亏比、持有期和个股贡献；
3. 增加历史成分股/可交易股票池，控制生存者偏差；
4. 把参数扰动、样本外和成本压力测试建模为批量实验，而不是单次回测结论；
5. 为回测增加不可变的运行编号，长期保留多版本结果并支持版本对比。
